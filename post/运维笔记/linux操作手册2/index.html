nginx配置重定向和转发,跨域问题解决

配置如下

server {
        listen       8090;
        server_name  127.0.0.1;
        location /{
              root /home/tomcat/font/dist;
              index index.html login.html login.htm;
              try_files $uri $uri/ /index.html;
              add_header 'Access-Control-Allow-Origin' '*';
              add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
              add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range';
              add_header 'Access-Control-Expose-Headers' 'Content-Length,Content-Range';
 
        }
 
        location ^~/server_context/history/ {
              rewrite ^/server_context/history/(.*) /server_context/$1 break;
              proxy_pass http://127.0.0.1:8030;
        }
 
          location ^~/server_context/ {
              proxy_pass http://127.0.0.1:8040/server_context/;
            
        }
}

Docker 操作指南

docker-compose -f docker-compose-peer.yaml up -d

1.停止所有的container，这样才能够删除其中的images：

docker stop $(docker ps -a -q)

如果想要删除所有container的话再加一个指令：

docker rm $(docker ps -a -q)

连停带删一起来 docker rm -f $(docker ps -aq)

查看docker容器： docker ps -a

重启docker容器： docker restart c4f5e1aa05a8

进入docker里面： docker exec -it c4f5e1aa05a8 /bin/bash

2.查看当前有些什么images

docker images

3.删除images，通过image的id来指定删除谁

docker rmi

想要删除untagged images，也就是那些id为的image的话可以用

docker rmi $(docker images | grep “^” | awk “{print $3}”)

要删除全部image的话

docker rmi $(docker images -q)

LNMP/Nginx常用Rewrite(伪静态规则)

相信现在大部分用Linux VPS的朋友都在使用这个迅速传播的Nginx或LNMP一键安装包（lnmp安装包已经自带了下面大部分伪静态规则），Nginx和Apache的伪静态是不一样的，今天就整理一下最常见的PHP程序的nginx Rewrite(伪静态规则)。目前已收集Wordpress、Wordpress二级目录、Discuz、Discuz X、PHPWind、Typecho、codeiginiter/thinkphp/laravel/yii2框架、drupal、DEDECMS、PHPCMS、ECSHOP、ShopEX、SaBlog： WordPress：

location / {
try_files $uri $uri/ /index.php?$args;
}

# Add trailing slash to */wp-admin requests.
rewrite /wp-admin$ $scheme://$host$uri/ permanent;
