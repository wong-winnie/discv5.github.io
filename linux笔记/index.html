    重定向和转发,跨域问题解决¶

    配置如下

    server {
        listen       8090;
        server_name  127.0.0.1;
        location /{
              root /home/tomcat/font/dist;
              index index.html login.html login.htm;
              try_files $uri $uri/ /index.html;
              add_header 'Access-Control-Allow-Origin' '*';
              add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
              add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range';
              add_header 'Access-Control-Expose-Headers' 'Content-Length,Content-Range';

        }

        location ^~/server_context/history/ {
              rewrite ^/server_context/history/(.*) /server_context/$1 break;
              proxy_pass http://127.0.0.1:8030;
        }

          location ^~/server_context/ {
              proxy_pass http://127.0.0.1:8040/server_context/;

        }
}

    git操作命令¶

    基础命令操作¶

    ssh-keygen #产生公私钥 git status #查看状态 git branch -a #查看分支列表 git pull #拉取代码 git reset --hard #回滚本地未提交的修改 git reset --hard 665424f7dbf #回滚到指定版本 git checkout -b fea_clock #创建新分支并切换 git add -A / . #添加全部修改 git commit -m "add" #提交修改 git push origin master #提交到master分支 git checkout fea_clock #切换到fea_clock分支 git merge fea_clock git checkout a.cpp #回退修改的文件 git cherry-pick 665424f7dbf # 合并某个提交 git remote -v #查看当前git路径

    删除远程分支¶

    git rebase --abort # 终止rebase错误 git push origin :fea_bug git remote prune origin git branch -D fea_bug

    合并分支：¶

    git branch -a git checkout fea_clock git add . git commit -m "update"

    git rebase develop 或者 git rebase -i 665424f7dbf

    git checkout develop git merge fea_clock git push origin develop

    快捷键命令¶

    git config --global alias.tr "log --graph --decorate --pretty=format:'%h -%C(red)%d%Creset %s (%Cblue%cn%Creset %ci -- %cr)' --abbrev-commit"

    Git自动补全设置¶

    下载git-completion.bash 并放到home 目录下：

    curl https://raw.githubusercontent.com/git/git/master/contrib/completion/git-completion.bash -o ~/.git-completion.bash

    将如下行放到.bash_profile 文件中

    if [ -f ~/.git-completion.bash ]; then . ~/.git-completion.bash fi

    将其立即生效： source ~/.bash_profile

    此时就可以代码补全了，比如我要切换到某个branch，则只需要输入git branch 然后按「TAB」键即可选则需要的branch分支了。

    docker操作指南¶

    docker-compose -f docker-compose-peer.yaml up -d

    1.停止所有的container，这样才能够删除其中的images：

    docker stop $(docker ps -a -q)

    如果想要删除所有container的话再加一个指令：

    docker rm $(docker ps -a -q)

    连停带删一起来 docker rm -f $(docker ps -aq)

    查看docker容器： docker ps -a

    重启docker容器： docker restart c4f5e1aa05a8

    进入docker里面： docker exec -it c4f5e1aa05a8 /bin/bash

    2.查看当前有些什么images

    docker images

    3.删除images，通过image的id来指定删除谁

    docker rmi

    想要删除untagged images，也就是那些id为的image的话可以用

    docker rmi $(docker images | grep "^" | awk "{print $3}")

    要删除全部image的话

    docker rmi $(docker images -q)

    sudo docker export 7691a814370e > ubuntu.tar
